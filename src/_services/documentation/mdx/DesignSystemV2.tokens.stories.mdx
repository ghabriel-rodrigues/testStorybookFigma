# Design Tokens

Os design tokens são nossas variáveis semânticas de estilo, propriedades que conseguem expressar 
um significado comum entre os times de Engenharia, Produto e Design para designar informações referentes 
a aparência de um respectivo componente. Com eles, definimos cores, espaçamentos, bordas, fontes e breakpoints.

Nesse momento, as versões dos tokens que existem no DS2 foram criadas a partir da perspectiva do 
[DS1](https://design-system.app.internal.gupy.io/design-tokens) e são mantidas utilizando a lib
[Amazon Style Dictionary](https://amzn.github.io/style-dictionary/#/).

Aqui neste documento você encontrará mais informações sobre os tipos de tokens e mais abaixo, 
existe uma seção auxiliando com as dúvidas mais frequentes das pessoas desenvolvedoras em relação a esse tema.

## Cores

As definições de cores do Design System da Gupy tem passado por mudanças para tornar os produtos Gupy mais inclusivos e acessíveis
para pessoas com diferentes níveis de percepção visual.

Provisoriamente, devido a versão do DS2 ser MVP, as cores estão sendo criadas com a sigla `DS` atrelada ao nome, 
dada a possível necessidade de suportarmos diferentes versões dos tokens por diferentes versões do DS. Futuramente deverão ser lançadas novas releases, 
alterando os nomes dos tokens conforme o time de Design fizer essas definições.

**As cores podem ser encontradas nos dois formatos abaixo:**

- `scss` - `$ds-color-primary-blue-dark`
- `typescript` - `DsColorPrimaryBlueDark` 

**Exemplo de uso com Typescript:**
<Source
  language='js'
  dark
  format={false}
  code={dedent`
  import styled, { css } from 'styled-components'
  import { DsColorPrimaryBlueDark } from '@/tokens/tokens'
  
  const Input = styled.input\`
    background: \${DsColorPrimaryBlueDark};\`
  `}
/>

## Perguntas frequentes

**Como gerar os tokens caso eu tenha clonado o DS2 para o meu computador?**

O comando que gera tokens é o `npm run tokens` e ele precisa ser executado na raíz do projeto.


**Onde ficam os Design Tokens do DS2 que podem ser utilizados?**

Os design tokens hoje possuem dois diferentes arquivos de uso:
* `scss`, se localizam no arquivo `src/tokens/tokens.scss`
* `consts`, se localizam no arquivo `src/tokens/tokens.ts`

IMPORTANTE: Nunca acrescente tokens diretamente nesses arquivos, eles são gerados sempre a
partir da lib Style Dictionary.

**Como posso editar ou acrescentar um novo token ao DS2?**

Para você acrescentar um novo token você pode acessar a estrutura de arquivos localizada em
`src/_services/tokens/properties/ds/` , verificando qual atributo você gostaria de editar 
ou acrescentar a partir do nome das pastas. 

Dentro dela você poderá perceber que temos:

* `border/` 
* `color/` 
* `font/` 
* `screen/` 
* `spacing/` 
* `transition/` 

Abaixo você pode ver um exemplo desses arquivos, no caso o referente à geração de bordas: `src/_services/tokens/properties/ds/border/main.json` 

<Source
  language='json'
  dark
  format={false}
  code={dedent`
     {
      "ds": {
        "border": {
          "radius": {
            "value": "4px",
            "attributes": { "category": "size", "type": "radius" }
          },
          "width": {
            "value": "1px",
            "attributes": { "category": "size", "type": "border" }
          }
        }
      }
    }
  `}
/>
