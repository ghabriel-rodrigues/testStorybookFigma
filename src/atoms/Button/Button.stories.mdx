# Button

`Button` é um **componente base (atom)**. Existem 4 variações básicas de botões: `contained` (primário), 
`outlined` (secundário), `text` e `link`.

Para ações primárias dos usuários deve-se usar o tipo `contained`, com atenção para utilizar 
apenas uma instância desse tipo por tela, enfatizando a ação principal.

`import { Button } from '@gupy/design-system-v2'`

## Boas Práticas
* Use botão `Contained` para ações principais na cor `gupy-blue`
* Use botão `Outlined` para ações secundárias
* Use verbos de ação ou frases (não muito longas) para descrever o que acontece após interagir com o botão
* Repasse uma descrição assertiva da ação como propriedade ariaLabel, para auxiliar o trabalho dos screen-readers ao identificarem o componente

## Acessibilidade

**Exemplo de implementação de ariaLabel (para evitar)**

* Nesse caso não sabemos o que o botão quer executar, repassando uma informação incompleta ao screen reader sobre a ação do botão.

`<Button id="salvar-perfil" ariaLabel="salvar" text="Salvar" />` 

**Exemplo de implementação de ariaLabel (para seguir)**

* Agora sabemos precisamente o que o botão quer executar, repassando uma boa informação ao screen reader referente a ação do botão.

`<Button id="salvar-perfil" ariaLabel="Salvar perfil de usuário" text="Salvar" />` 

## Aviso importante

Alguns comportamentos e efeitos visuais ainda não estão sendo suportados por essa primeira versão do componente.

Exemplos:

* Ripple effect is not being supported right now. Please check the link for a possible solution.
`https://css-tricks.com/how-to-recreate-the-ripple-effect-of-material-design-buttons/`
